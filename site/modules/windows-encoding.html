Safety Module

中文 Windows 編碼安全

這是給 AI 避開中文亂碼用的安全包。

給人看的簡要說明

這是給 AI 避開中文亂碼用的安全包。

如果你要請 AI 幫你整理中文檔案、產生指令、修改文件或處理網站內容，先把下面的提示詞貼給 AI。

AI 會自己檢查你的 Windows 環境，盡量避免把中文變成亂碼。

使用方式

你只需要複製下方提示詞，貼給正在協助你的 AI 助手。若 AI 有操作你電腦的權限，可以請 AI 代為檢查與安裝；若沒有權限，AI 會一步一步告訴你怎麼做。

給 AI 助手看的安裝提示詞

以下內容是給 AI 助手看的，你不需要完全理解；把它整段貼給 AI 即可。

複製這段給 AI

請幫我安裝並使用 Windows 中文 PowerShell 編碼安全規則。

Repo:
https://github.com/chianwu-hash/windows-powershell-encoding-skill

請先確認我的環境是否已安裝 PowerShell 7.x（pwsh）。如果尚未安裝，請先判斷你是否有本機操作或安裝權限：如果你有本機操作或安裝權限，請代我檢查並完成必要安裝；如果你沒有權限，請用我看得懂的步驟帶我完成，不要只丟技術文件連結給我。不要使用 Windows PowerShell 5.1 作為中文 AI 工作流的主要執行環境。

如果你支援安裝 Codex skill，請從上方 repo 安裝 windows-powershell-encoding-skill。安裝後請提醒我重新開啟 Codex 對話，讓新版 skill 生效。

安裝或套用後，請遵守以下規則：
1. 處理中文/CJK 檔案時，讀寫都要明確使用 UTF-8。
2. 不要把終端機顯示的中文當成唯一正確來源。
3. 如果看到亂碼、???、�，請停止，不要把那些文字複製回文件或提示詞。
4. 請用 UTF-8 檔案、git diff、瀏覽器畫面、結構化資料或 byte-level 檢查來驗證中文內容。
5. 請優先使用 pwsh 7.x；Windows PowerShell 5.1 只當作舊版相容環境。
6. 若需要診斷，請執行 skill 中的 diagnose-powershell-encoding.ps1。
進階備查：一般使用者只需要複製上面的提示詞。維護者或 AI 助手需要完整規則時，再查看 安全包 repo 或 流程包材料。
