# 行政案件類型路由流程

## 流程總覽

```text
收到案件
-> 判斷案件類型
-> 讀取對應規則
-> 檢查禁止誤用
-> 產生對應輸出
-> 高風險時等待人工確認
```

## 步驟 1：列出案件類型

先列出最常見的行政案件，例如：

- 場地借用
- 會議紀錄
- 活動通知
- 行事曆操作
- 一般行政來文

## 步驟 2：為每類案件指定規則

每類案件要有：

- 必讀規則
- 禁止誤用
- 預設輸出
- 何時需要人工確認

## 步驟 3：建立路由表

使用 `02-template/case-routing-table-template.md`。

## 步驟 4：用假案例測試

用 `03-demo-data/sample-cases.md` 練習分類。

## 步驟 5：修正規則

如果 AI 分類錯誤，補上：

- 更明確的判斷條件
- 禁止誤用
- 人工確認條件
